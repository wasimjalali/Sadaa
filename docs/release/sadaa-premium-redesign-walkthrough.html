Language Memory

The old dictionary is replaced by a stronger system: terms, pronunciations, aliases, priority, language targeting, usage-aware biasing, pause/resume controls, live-preview deterministic replacements, deterministic snippets, suggestions, and history memory/snippet-hit diagnostics. Raw-mode dictation still applies local deterministic memory without calling GPT.

1.
Add a term like Karko AI with priority, language, and sounds-like hints.
2.
Add a replacement like cloud code → Claude Code, then pause experiments without deleting them.
3.
Create snippets for repeated spoken phrases; they expand before/after formatting, in raw mode, and when the formatter falls back.
4.
Copy/import JSON and CSV, then see memory/snippet hits in History and used-count badges in Language Memory.

Notes are now a local workspace with pinned notes, tags, auto-save drafts, word and character stats, markdown copy, JSON backup/restore, duplication, and append-latest-dictation.

CaptureFocus the editor and dictate directly into a note.

History Actions

History rows now expose the recovery actions that matter during real use.

CopyQuickly restore any past dictation.
Send To ScratchpadMove useful text into notes without re-dictating.
ReprocessRe-run retained audio when available, then apply current Language Memory.
Learn CorrectionTurn a bad output into a term or replacement rule.

Model Guidance

FastUse your gpt-4o-mini-transcribe deployment, or the newest mini transcribe variant your region exposes.
AccurateUse gpt-4o-transcribe when maximum quality matters.
Speech/MAIUse mai-transcribe-1.5 where your Azure Speech resource supports it.
RealtimeKeep realtime audio models for future streaming workflows; Sadaa's current hotkey flow is file-based transcription.
LegacyUse whisper only as a compatibility fallback.
